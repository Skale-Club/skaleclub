## Spacing Scale

Standard 8-point scale. The viewer uses full-viewport sections — intra-section content spacing follows these tokens.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Dot gap inactive, inline icon spacing |
| sm | 8px | `gap-2`, `p-2` | Feature list item gap, badge padding |
| md | 16px | `gap-4`, `p-4` | Card content vertical rhythm, input padding |
| lg | 24px | `gap-6`, `p-6` | Card-to-card within a section, gate form gap |
| xl | 32px | `gap-8`, `p-8` | Navigation dot column gap from viewport edge |
| 2xl | 48px | `py-12` | Section content max-width padding on small screens |
| 3xl | 64px | `px-16` | Section content horizontal padding on wide viewports |

Exceptions:
- Navigation dots: 8px gap between dots (`gap-2`), active dot `w-2.5 h-2.5` (10px), inactive dot `w-2 h-2` (8px) — sub-4px precision is acceptable for decorative indicator sizing
- Touch target minimum: each navigation dot button has `min-w-[44px] min-h-[44px]` container even though the visual dot is smaller — ensures WCAG 2.5.5 touch target compliance
- Section content card: `max-w-xl` (576px) centered with `px-6` horizontal padding on mobile, `px-16` on large screens

## Typography

The viewer uses the same Inter font as the rest of the project. Two weights only — semibold for display/service titles, regular for all body and supporting text.

| Role | Size | Tailwind Class | Weight | Tailwind Weight | Line Height | Usage |
|------|------|----------------|--------|-----------------|-------------|-------|
| Display | 48px | `text-5xl` | Semibold | `font-semibold` (600) | 1.1 | Cover section — client name (`clientName`) |
| Heading | 28px | `text-3xl` | Semibold | `font-semibold` (600) | 1.2 | Service section title, closing section headline |
| Body | 16px | `text-base` | Regular | `font-normal` (400) | 1.5 | Service description, intro body text, features list items |
| Label | 14px | `text-sm` | Regular | `font-normal` (400) | 1.4 | Price display, "Proposal for" eyebrow label, section metadata, gate heading, inline error |

Weights in use: **Regular (400)** and **Semibold (600)**. Exactly 2 weights.

**Special treatment:**
- "Proposal for" eyebrow on cover: `text-sm font-normal tracking-widest uppercase text-zinc-400`
- Price value on service sections: `text-2xl font-semibold text-white` — uses heading size to give price visual emphasis
- Feature list items: `text-sm font-normal text-zinc-300` — slightly dimmer than body to reduce visual noise

## Color

The viewer is always rendered in a hardcoded dark palette using Tailwind zinc utilities. It does NOT use HSL CSS variables. Colors are static regardless of light/dark mode.

| Role | Value | Tailwind Token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#09090b` | `bg-zinc-950` | Main viewer container background (D-01) |
| Secondary (30%) | `#18181b` | `bg-zinc-900` | Section gradient overlay base, access code gate input background, service card background |
| Accent (10%) | `#ffffff` | `text-white` / `bg-white` | Reserved — see list below |
| Error / Destructive | `hsl(var(--destructive))` | `text-destructive` | Inline error under access code input ("Incorrect code") |

**Accent (white) reserved for:**
1. Active navigation dot — `bg-white scale-125`
2. Client name on cover section — `text-white text-5xl font-semibold`
3. Service section title — `text-white text-3xl font-semibold`
4. Closing section headline — `text-white`
5. Access code gate "Submit" button — uses `Button` default variant (primary blue, existing shadcn token) — NOT pure white. See note below.

**Supporting colors:**
- `text-zinc-400` — dimmed supporting text (eyebrow labels, intro body, section subtitles)
- `text-zinc-300` — feature list items (slightly less dim than zinc-400, more readable at scale)
- `bg-white/30` — inactive navigation dot (30% opacity white)
- `bg-white/60` — navigation dot hover state (60% opacity white)
- `border-zinc-700` — access code input border (visible but subtle against zinc-950)
- Section gradient: `bg-gradient-to-b from-zinc-800/20 to-transparent` — from zinc-800 at 20% opacity fading to transparent (D-02)

**Access code button note:** The "Submit" button in the gate uses the standard shadcn `Button` component at `variant="default"` — which resolves to `--primary` (brand blue `hsl(217 91% 60%)` in dark context). This maintains brand consistency even inside the dark viewer. Do NOT override to white or zinc.

**Admin list badges (additive changes to EstimatesSection.tsx):**
- View count badge: `Badge variant="secondary"` with `Eye` icon + count integer — inherits admin theme colors
- Last-seen text: `text-xs text-muted-foreground` — no badge, inline text

## Interaction Contracts

### View Tracking (EST-11, D-12)

- Pattern: `useMutation` + `useRef(false)` guard — exact VCard.tsx pattern
- Fires: Once after `estimateData` is available and access code is bypassed (if applicable)
- Endpoint: `POST /api/estimates/:id/view`
- No UI feedback on success or failure — fire-and-forget
- Guard: `hasTrackedView.current = true` set before mutation fires, preventing double-fire in StrictMode

### Access Code Verification (EST-12, D-07–D-09)

- Gate shown: when `data.hasAccessCode === true` and `isUnlocked === false`
- Submit: `POST /api/estimates/:id/verify-code` with `{ code }`
- On 200: set `isUnlocked = true` (React state) — viewer renders, view tracking fires
- On 401: set inline `error = "Incorrect code"` — error shown under input, input retains focus
- On Enter key: same behavior as clicking Submit button
- Code field: clears error state on any `onChange` event
- Submit button: disabled when `!code` or `isPending`

### Navigation Dot Click

- `scrollIntoView({ behavior: 'smooth' })` on the target section ref
- Do NOT use `scroll-behavior: smooth` globally — only per individual dot click
- Active dot tracked via `IntersectionObserver` at threshold `0.5` on each section ref

### Framer-motion Section Entrance

- `viewport={{ once: true }}` — animation fires only on first intersection, not on re-scroll
- Applied to section content wrapper, not to the section element itself (section is the snap target)

### Admin List Badges (EstimatesSection.tsx additive changes, D-13)

- View count badge: rendered as `<Badge variant="secondary">` with `<Eye w-3 h-3 />` + integer count
- Count of 0 displays as "0" — do not hide at zero (absence of views is informative)
- Last seen text: `formatDistanceToNow(new Date(est.lastViewedAt), { addSuffix: true })` — shown only when `est.lastViewedAt !== null`
- Both badges appear inline in the list row, to the right of the creation date, left of the action buttons
- Access code field in dialog: plain `Input` type="text", label "Access code (optional)", placeholder "e.g. 20260419", helper text "text-xs text-muted-foreground" — "Leave blank to disable gate"
