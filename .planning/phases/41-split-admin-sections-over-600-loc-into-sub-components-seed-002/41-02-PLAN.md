<objective>
Split `client/src/components/admin/integrations/AIAssistantCard.tsx` (643 LOC) into the card shell plus three per-provider tab components. The card retains the tab strip and provider switching; each tab body becomes its own self-contained file that owns its own provider state, queries, and mutations.

Purpose: get under 600 LOC per file; each provider is independently testable and clearer to read.
Output: 1 refactored shell + 3 new tab files.
</objective>

<context>
@CLAUDE.md
@client/src/components/admin/integrations/AIAssistantCard.tsx

The current file holds three nearly-symmetric provider blocks, each with their own:
- local state (settings draft, API key input, testing/saving flags, test result enum + message)
- React Query (settings GET)
- save function
- toggle handler
- test connection function
- JSX panel (lines 463-506 for OpenAI, 508-554 for Gemini, 556-638 for OpenRouter)

The shared piece is the tab strip (lines 429-457) that calls `switchActiveProvider` and reads each tab's `enabled` flag for the ON/OFF pill.

Approach: each tab component MANAGES ITS OWN STATE for its provider. The shell uses a callback to receive `enabled` for the ON/OFF pill (so the strip can stay accurate) — pass an `onEnabledChange(enabled: boolean) => void` prop down, called from each tab's settings load + toggle handler.
</context>
