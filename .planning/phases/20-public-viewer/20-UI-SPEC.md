# Phase 20 — UI Design Contract: Public Viewer

> Visual and interaction contract for the `/p/:slug` fullscreen bilingual presentation viewer.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | shadcn/ui | components.json |
| Style | new-york | components.json |
| Preset | neutral base color, CSS variables | components.json |
| Component library | Radix UI (via shadcn) | components.json |
| Icon library | lucide-react | existing codebase |
| Heading font | Outfit (loaded in index.html via Google Fonts) | CLAUDE.md + index.html |
| Body font | Inter (--font-sans in index.css) | index.css |

**Note:** `--font-display` is currently set to `'Inter'` in index.css even though Outfit is loaded in the HTML. For PresentationViewer, headings MUST use `font-family: 'Outfit', sans-serif` via inline style or a `font-outfit` utility class — not `font-display` — until the CSS variable is corrected upstream.

## Route Isolation

The viewer route `/p/:slug` is fully isolated from the standard site layout.

| Suppressed element | Method |
|--------------------|--------|
| Navbar | `isPresentationRoute` guard in App.tsx (symmetric to `isEstimateRoute` pattern) |
| Footer | Same guard |
| ChatWidget | Same guard |

`isPresentationRoute` definition: `location.startsWith('/p/')`.

The App.tsx isolated branch renders only `<Route path="/p/:slug" component={PresentationViewer} />` with no surrounding shell components.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Bullet item gap, icon-to-text inline gap |
| sm | 8px | Label-to-heading gap (uppercase eyebrow above heading) |
| md | 16px | Body text margin-top, list item spacing |
| lg | 24px | Heading-to-body gap within a slide |
| xl | 32px | Content block padding on mobile (px-8 = 32px) |
| 2xl | 48px | Vertical padding inside access gate form |
| 3xl | 64px | max-w content column (max-w-xl = 576px — not a spacing token but consistent with EstimateViewer) |

Exceptions:
- Navigation dot touch targets: 44px × 44px minimum (wrapping `<button>` with `min-w-[44px] min-h-[44px]`) — mirrors EstimateViewer pattern exactly.
- Language switcher touch target: 44px height minimum.

## Typography

The viewer surface is full dark (`bg-zinc-950`). All sizes below refer to rendered slide content.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 16px | 400 (regular) | 1.625 (leading-relaxed) | Inter | Slide body text, bullet text, gate form labels |
| Label | 12px | 400 (regular) | 1.6 (leading-normal) | Inter | Uppercase eyebrow labels (`tracking-widest uppercase text-zinc-400`) |
| Heading | 30px (text-3xl) | 600 (semibold) | 1.2 | Outfit | Slide heading for `title-body`, `bullets`, `two-column`, `stats`, `image-focus`, `section-break`, `closing` |
| Display | 48px (text-5xl) | 600 (semibold) | 1.1 (leading-tight) | Outfit | Cover slide primary heading only (`cover` layout) |

**Stats layout exception:** stat numbers render at 48px / weight 700 (bold) to communicate numerical impact — this is a deliberate visual hierarchy departure from the standard heading scale.

## Color

The PresentationViewer is a standalone dark-mode surface. It does NOT inherit the site's CSS variable theme (no `.dark` class toggling needed — always dark).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` (zinc-950) | Full-screen slide background (`bg-zinc-950`) |
| Secondary (30%) | `rgba(255,255,255,0.06)` (zinc-800/20 gradient overlay) | Subtle gradient overlay per section (`bg-gradient-to-b from-zinc-800/20 to-transparent`) |
| Foreground primary | `#ffffff` | All heading text, active navigation dots, access code inputs |
| Foreground secondary | `zinc-400` (`rgb(161 161 170)`) | Eyebrow labels, body text, inactive nav dots, form helper text |
| Accent (10%) | `#1C53A3` (Primary Blue) | Language switcher active state indicator, access code submit button background |
| CTA (Brand Yellow) | `#FFFF01` | "Unlock Presentation" CTA button text background — pill-shaped, black bold text |

Accent (`#1C53A3`) reserved for:
1. Language switcher active-language indicator pill
2. Access code "Unlock Presentation" button hover/active ring

Brand Yellow (`#FFFF01`) reserved for:
1. Primary CTA button: "Unlock Presentation" in AccessCodeGate

## Component Inventory

These are the components that make up the viewer. All are either already installed shadcn components or native HTML/framer-motion elements.

### PresentationViewer (root)
- `h-screen overflow-y-scroll snap-y snap-mandatory bg-zinc-950 text-white`
- No Navbar, Footer, or ChatWidget.
- Contains: navigation dots (fixed right), language switcher (fixed top-right), slide sections.

### Navigation Dots (fixed sidebar)
- `fixed right-4 top-1/2 -translate-y-1/2 flex flex-col gap-2 z-50`
- One `<button>` per slide: `min-w-[44px] min-h-[44px] flex items-center justify-center`
- Active dot: `w-3 h-3 bg-white rounded-full scale-125`
- Inactive dot: `w-2 h-2 bg-white/30 hover:bg-white/60 rounded-full`
- Click scrolls to section: `sectionRefs.current[i]?.scrollIntoView({ behavior: 'smooth' })`

### Language Switcher (fixed top-right)
- `fixed top-4 right-16 z-50` (left of nav dots column)
- Renders two clickable labels: `EN` and `PT`
- Active label: `text-white font-semibold text-sm`
- Inactive label: `text-zinc-500 hover:text-zinc-300 text-sm cursor-pointer`
- State stored in URL query param: `?lang=en` (default) or `?lang=pt-BR`
- Language change via `useLocation` / Wouter `setSearch` — does NOT reload page, does NOT reset scroll position.
- NOT using the global `LanguageToggle` component (that component uses global `useTranslation` context which is scoped to the main site). PresentationViewer reads `lang` from URL search params only.

### Slide Section (per SlideBlock)
- `h-screen w-full snap-start relative flex items-center justify-center`
- Inner content: `relative z-10 px-8 max-w-xl mx-auto w-full`
- Gradient overlay (all slides): `absolute inset-0 bg-gradient-to-b from-zinc-800/20 to-transparent pointer-events-none`
- framer-motion animation: `initial={{ opacity: 0, y: 20 }}`, `whileInView={{ opacity: 1, y: 0 }}`, `transition={{ duration: 0.5, ease: 'easeOut' }}`, `viewport={{ once: true }}`

### SlideBlock Layout Variants (8 total)

All variants share the section wrapper above. Inner layout differs:

| Layout | Inner Structure |
|--------|----------------|
| `cover` | Centered: eyebrow label (text-zinc-400) → Display heading (text-5xl Outfit semibold) → optional subheading (text-zinc-400 text-sm mt-6) |
| `section-break` | Centered: eyebrow label → Heading (text-3xl Outfit semibold) → optional body (text-zinc-400 text-base leading-relaxed) |
| `title-body` | Left-aligned: eyebrow label → Heading (text-3xl) → body (text-base text-zinc-400 leading-relaxed mt-4) |
| `bullets` | Left-aligned: eyebrow label → Heading (text-3xl) → `<ul>` with `space-y-3`; each bullet: `flex gap-2` with `–` separator in zinc-500 + `text-sm text-zinc-300` |
| `stats` | Left-aligned or centered: eyebrow label → `<dl>` grid (grid-cols-2 gap-8); stat number `text-5xl font-bold text-white`; stat label `text-sm text-zinc-400 mt-1` |
| `two-column` | Two equal columns (grid-cols-2 gap-16): left col has heading, right col has body text |
| `image-focus` | Image fills upper 50% of section (object-cover) with gradient overlay; heading + body in lower 50% |
| `closing` | Centered: eyebrow label → Heading (text-3xl) → body (text-zinc-400 text-sm mt-2) |

**Bilingual field resolution rule (all layouts):**
```
lang === 'pt-BR'
  ? (slide.headingPt || slide.heading)   // fallback to EN if PT empty
  : slide.heading

lang === 'pt-BR'
  ? (slide.bodyPt || slide.body)
  : slide.body

lang === 'pt-BR'
  ? (slide.bulletsPt?.length ? slide.bulletsPt : slide.bullets)
  : slide.bullets
```

## Interaction Contracts

### Scroll-snap behavior
- Container: `overflow-y-scroll snap-y snap-mandatory` on the root `div`
- Each section: `snap-start` — one slide snaps to fill the full viewport
- No horizontal scroll; no mobile momentum issues (snap-mandatory handles this)

### Language switching
- `useLocation` hook (Wouter) reads `?lang=` search param on mount
- Default: `en` (no param or `?lang=en`)
- Switch: updates URL search param in-place via `navigate` with `{ replace: true }` — no scroll reset, no full re-mount
- Component reads active language from local derived state: `const lang = searchParams.get('lang') === 'pt-BR' ? 'pt-BR' : 'en'`

### View tracking
- Tracked once per page load via `useRef(false)` guard
- Fires `POST /api/presentations/slug/:slug` or a dedicated view endpoint only AFTER access gate is passed (if applicable)
- Implementation mirrors EstimateViewer's `hasTrackedView` ref pattern exactly

### IntersectionObserver for active slide
- Threshold: 0.5 (slide must be 50% visible to become active)
- Updates `activeIndex` state which drives the active navigation dot style
- Observer disconnected on cleanup

### framer-motion slide entrance
- `initial={{ opacity: 0, y: 20 }}`
- `whileInView={{ opacity: 1, y: 0 }}`
- `transition={{ duration: 0.5, ease: 'easeOut' }}`
- `viewport={{ once: true }}` — animates only on first entrance, not on scroll-back

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Navigation dot touch target | `min-w-[44px] min-h-[44px]` wrapper button |
| Navigation dot label | `aria-label="Go to slide {n+1}"` |
| Language switcher | `aria-label` on each clickable label: "Switch to Portuguese" / "Switch to English" |
| Access code input | `<label>` visually hidden or `aria-label` on Input |
| Access code submit on Enter | `onKeyDown` handler on Input |
| Spinner | `aria-label="Loading"` on Loader2 |
| Color contrast | All body text: zinc-400 on zinc-950 — 4.6:1 (AA compliant); white headings on zinc-950 — 18:1 (AAA) |
