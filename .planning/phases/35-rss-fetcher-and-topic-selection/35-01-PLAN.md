<objective>
Build the RSS ingestion engine: a single `fetchAllRssSources()` entry point that iterates every enabled source, parses its feed via `rss-parser`, upserts items by GUID through the existing storage layer, and records per-source success/error state.     export interface FetchSummary {
      sourcesProcessed: number;
      itemsUpserted: number;
      errors: Array<{ sourceId: number; sourceName: string; message: string }>;
    }

    export async function fetchAllRssSources(): Promise<FetchSummary> { ... }

    **Helpers (file-local, not exported):**

    1. `stripHtml(input: string | null | undefined): string` (D-06)
       - Returns `""` if input is null/undefined/empty.
       - Strip tags with `input.replace(/&lt;[^&gt;]*&gt;/g, " ")`.
       - Decode the most common entities: `&amp;amp; &amp;lt; &amp;gt; &amp;quot; &amp;#39; &amp;nbsp;` -> their literals (a tiny lookup map; do NOT pull in a new HTML-entity dep).
       - Collapse whitespace via `.replace(/\\s+/g, " ").trim()`.
       - Caller is responsible for `.slice(0, MAX_SUMMARY_CHARS)`.

    2. `resolveGuid(item: Parser.Item, sourceId: number): string` (D-05)
       - First non-empty wins:
         1. `item.guid` (rss-parser unifies `<guid>` and Atom `<id>` into `item.guid`).
         2. `item.link`.
         3. `createHash("sha256").update(\`${sourceId}|${item.title ?? ""}|${item.pubDate ?? item.isoDate ?? ""}\`).digest("hex")`.
       - Always returns a non-empty string. The (source_id, guid) UNIQUE index is the dedup truth (D-13).

    3. `parsePublishedAt(item: Parser.Item): Date | null`
       - Try `item.isoDate` first (rss-parser provides ISO 8601 when possible), fall back to `item.pubDate`.
       - Return `new Date(value)` if `Number.isFinite(date.getTime())`, else `null`.

    4. `processSource(source: BlogRssSource): Promise<{ upserted: number }>`
       - Parses the feed: `const feed = await parser.parseURL(source.url);`
       - Slices to `MAX_ITEMS_PER_SOURCE` most recent items (rss-parser returns items in feed order, which is normally newest-first — do NOT re-sort defensively; respect feed order).
       - For each item:
         - `const publishedAt = parsePublishedAt(item);`
         - **Skip if D-04 filter excludes:** if `publishedAt &amp;&amp; source.lastFetchedAt &amp;&amp; publishedAt &lt; source.lastFetchedAt` -> continue (do NOT upsert; the item predates our last successful sweep).
         - Build `InsertBlogRssItem`:
           ```typescript
           const guid = resolveGuid(item, source.id);
           const url = item.link ?? guid; // schema requires .url(); fallback to guid only if guid happens to be a URL — otherwise skip the item rather than violating the URL Zod check
           if (!isHttpUrl(url)) continue; // tiny helper: returns /^https?:\/\//.test(url)
           const summary = stripHtml(item.contentSnippet ?? item.content ?? item.summary ?? "")
             .slice(0, MAX_SUMMARY_CHARS) || null;
           const title = (item.title ?? "(untitled)").slice(0, 1000);
           await storage.upsertRssItem({
             sourceId: source.id,
             guid,
             url,
             title,
             summary,
             publishedAt: publishedAt ?? null,
             // status omitted — schema default "pending" applies (D-04 schema)
           });
           ```
         - Track upserted count locally.
       - Return `{ upserted }`.

    5. `fetchAllRssSources()` orchestrator (D-02, D-07, D-14):
       - `const sources = (await storage.listRssSources()).filter((s) =&gt; s.enabled);` — D-14 sequential, in DB order.
       - Iterate sequentially (`for (const source of sources) { ... }`) — NOT `Promise.all`.
       - Per source, wrap `processSource(source)` in try/catch.
       - **On success:**
         ```typescript
         await storage.updateRssSource(source.id, {
           lastFetchedAt: new Date(),
           lastFetchedStatus: "ok",
           errorMessage: null,
         });
         ```
       - **On error (D-07):**
         ```typescript
         const message = (err instanceof Error ? err.message : String(err))
           .slice(0, MAX_ERROR_MESSAGE_CHARS);
         await storage.updateRssSource(source.id, {
           lastFetchedAt: new Date(),
           lastFetchedStatus: "error",
           errorMessage: message,
         });
         summary.errors.push({ sourceId: source.id, sourceName: source.name, message });
         // continue to next source — do NOT throw
         ```
       - Console log per source (parity with `[cron]` style): prefix `[rss-fetcher]`, e.g., `console.log(\`[rss-fetcher] source=${source.id} ${source.name} upserted=${upserted}\`);` and `console.error(\`[rss-fetcher] source=${source.id} ${source.name} error: ${message}\`);`
       - Return the aggregate `FetchSummary`.
