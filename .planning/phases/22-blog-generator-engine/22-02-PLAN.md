  <action>
Finish the implementation in `server/lib/blog-generator.ts` using the helper from `server/lib/blog-gemini.ts`.

Text generation requirements:
- call Gemini once to propose a single topic based on `seoKeywords`, `promptStyle`, and `enableTrendAnalysis`
- call Gemini a second time to return structured JSON with these exact fields: `title`, `content`, `excerpt`, `metaDescription`, `focusKeyword`, `tags`
- keep the model as `gemini-1.5-flash`
- parse JSON defensively and throw a clear error if required fields are missing
- treat `content` as HTML-ready text saved directly into `blog_posts.content`

Image requirements:
- call Gemini image generation with `gemini-2.0-flash-exp`
- if image bytes are returned, upload them through `getSupabaseAdmin()` to bucket `images` at path `blog-images/${Date.now()}-${randomUUID()}.jpg`
- set `contentType: "image/jpeg"` and `upsert: false`
- read the public URL with `supabase.storage.from("images").getPublicUrl(path)`
- if Gemini image generation fails, returns no image payload, or Supabase upload fails, `console.warn(...)` and continue with `featureImageUrl = null`

Draft post creation requirements:
- create the post with `storage.createBlogPost({ ... })`
- set `status: "draft"`
- set `authorName: "AI Assistant"`
- serialize `tags` as `tags.join(", ")`
- generate `slug` from the title with:

```typescript
const baseSlug = title
  .toLowerCase()
  .replace(/[^a-z0-9]+/g, "-")
  .replace(/^-+|-+$/g, "") || "blog-post";
const slug = `${baseSlug}-${Date.now()}`;
```

Completion ordering requirements:
- create the draft post first
- only after that succeeds, call `storage.updateBlogGenerationJob(job.id, { status: "completed", postId: post.id, completedAt: now, reason: null, error: null })`
- then clear the lock and update `lastRunAt` on the singleton settings row
- return `{ skipped: false, reason: null, jobId: job.id, postId: post.id, post }`

Failure requirements:
- if topic generation, content generation, JSON parsing, or post creation throws after the job exists, update the job to `failed`, set `completedAt`, store the error message, clear the lock, preserve the previous `lastRunAt`, and rethrow

Keep all route wiring out of this plan. The output of this plan is a callable generator service only.
