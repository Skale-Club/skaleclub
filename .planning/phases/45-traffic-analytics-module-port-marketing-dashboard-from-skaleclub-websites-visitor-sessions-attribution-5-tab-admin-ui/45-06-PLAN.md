(`MarketingSection.tsx`), 5 tab components, and 1 shared utils file. Wire the section
into the existing admin shell (sidebar entry, lazy import, slug map, section-list

1. **Replace `AdminPageHeader` with this codebase's `SectionHeader`** (from
   `@/components/admin/shared`). The source uses a different header component;
   destination has `SectionHeader` as the universal admin section header per
   CLAUDE.md "Admin Design System". Section-list adds 'traffic' to
   `sectionsWithOwnHeader` so the Admin.tsx auto-rendered SectionHeader is suppressed
   (the section renders its own per the source's design).

   COLD per the source), since CONTEXT.md says "All UI strings in English." The
   underlying numbers come from QUENTE/MORNO/FRIO enum values via the SQL `FILTER`
   clauses in plan 02 вЂ” the UI just renders whatever count comes back.

   translation system ONLY runs on the public site (`useTranslation` hook). The admin
   panel uses raw English strings throughout (see existing admin components for
   precedent вЂ” none wrap strings in `t()`). This avoids adding ~30 new entries to
   `translations.ts` and stays under the 600-line cap there.

`@/lib/queryClient` `apiRequest` signature exactly (verified вЂ” both projects share the
pattern). Source's `staleTime: 30_000` policies port verbatim.

deliverable of Phase 45.
