```tsx
function AnalyticsProvider({ children }: { children: React.ReactNode }) {
  const { data: settings } = useQuery<CompanySettings>({
    queryKey: ['/api/company-settings'],
  });
  const [location] = useLocation();

  useEffect(() => {
    if (settings) {
      initAnalytics({ ... });
    }
  }, [settings]);

  useEffect(() => {
    trackPageView(location);
  }, [location]);

  // Phase 45 — mount the marketing attribution lifecycle alongside GTM/GA4 tracking.
  // The hook has its own /admin guard via isAttributionIgnoredPath.
  useAttribution();

  return <>{children}</>;
}
```

**Why this location?**
- `AnalyticsProvider` runs once at app boot and re-renders only when `settings` or
  `location` change — same lifecycle as `useAttribution` needs.
- `useAttribution` internally calls `useLocation()` so re-renders are driven by Wouter
  navigation, which is what we want.
- Wrapping `Router` in `AnalyticsProvider` (already done at line 322-326) means the
  hook fires for every route, including the catch-all DynamicLanding. Admin routes
  skip thanks to the hook's internal `/admin` guard.
- Calling the hook here AVOIDS adding a separate `<AttributionProvider>` component,
  keeping the App.tsx tree shape unchanged.

**Note on SSR safety:** This codebase is fully SPA — `window` is always defined when
React mounts. The hook's `window.location.search` read at mount is safe.
