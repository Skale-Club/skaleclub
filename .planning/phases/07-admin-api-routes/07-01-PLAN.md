**Handler 1 — GET /api/estimates (EST-03, D-09):**
- Middleware: `requireAdmin`
- Body: call `storage.listEstimates()`, return `res.json(estimates)`
- Wrap in try-catch; catch returns `res.status(500).json({ message: (err as Error).message })`

**Handler 2 — POST /api/estimates (EST-04, D-04, D-05, D-16):**
- Middleware: `requireAdmin`
- Validation: `const bodySchema = insertEstimateSchema.omit({ slug: true })` then `bodySchema.safeParse(req.body)` — if fail: `res.status(400).json({ message: "Validation error", errors: parsed.error.errors })`
- Slug generation: `const slug = crypto.randomUUID()`
- Create: `const estimate = await storage.createEstimate({ ...parsed.data, slug })`
- Return: `res.status(201).json(estimate)`
- Wrap in try-catch; catch returns `res.status(400).json({ message: (err as Error).message })`

**Handler 3 — PUT /api/estimates/:id (EST-04, D-06, D-07, D-17, D-08):**
- Middleware: `requireAdmin`
- Validation: `const updateSchema = insertEstimateSchema.partial().omit({ slug: true })` then `updateSchema.safeParse(req.body)` — if fail: `res.status(400).json({ message: "Validation error", errors: parsed.error.errors })`
- Pre-flight 404 check: `const existing = await storage.getEstimate(Number(req.params.id)); if (!existing) return res.status(404).json({ message: "Estimate not found" })` — this is REQUIRED because `updateEstimate` returns `undefined` when row is missing (Drizzle `.returning()` pitfall)
- Update: `const estimate = await storage.updateEstimate(Number(req.params.id), parsed.data)`
- Return: `res.json(estimate)`
- Wrap in try-catch; catch returns `res.status(400).json({ message: (err as Error).message })`

**Handler 4 — DELETE /api/estimates/:id (EST-04):**
- Middleware: `requireAdmin`
- Call: `await storage.deleteEstimate(Number(req.params.id))`
- Return: `res.json({ success: true })` (matches portfolio.ts convention — NOT 204)
- Wrap in try-catch; catch returns `res.status(400).json({ message: (err as Error).message })`

**Handler 5 — GET /api/estimates/slug/:slug (EST-05, D-10, D-15):**
- No middleware (public endpoint)
- IMPORTANT: Register this route BEFORE any `/:id` pattern routes to avoid Express matching "slug" as an id value (defensive practice per research pitfall 2 — even though no GET /:id exists today)
- Lookup: `const estimate = await storage.getEstimateBySlug(req.params.slug)`
- If null: `return res.status(404).json({ message: "Estimate not found" })`
- Return: `res.json(estimate)`
- Wrap in try-catch; catch returns `res.status(500).json({ message: (err as Error).message })`
- Do NOT call `setPublicCache` — per D-15, estimate data contains client names and pricing

**Route registration order inside the function (slug endpoint first):**
1. `app.get("/api/estimates/slug/:slug", ...)` — public, no auth
2. `app.get("/api/estimates", requireAdmin, ...)` — list
3. `app.post("/api/estimates", requireAdmin, ...)` — create
4. `app.put("/api/estimates/:id", requireAdmin, ...)` — update
5. `app.delete("/api/estimates/:id", requireAdmin, ...)` — delete
