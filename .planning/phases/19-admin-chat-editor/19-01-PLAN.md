**PresentationEditor sub-component (co-located, before main export):**

- Actually, simplest approach: reuse the list query data. The editor receives the `presentation` object from the parent's list query (pass it as a prop instead of re-fetching). Change props to: `{ presentation: PresentationWithStats; onBack: () => void }`
- Local state: `[jsonText, setJsonText]` initialized from `JSON.stringify(presentation.slides, null, 2)` — use `useState(() => JSON.stringify(presentation.slides ?? [], null, 2))`
- `useEffect` keyed to `presentation.id`: reset `jsonText` and `parsedSlides` when `presentation.id` changes (handles re-open after back navigation). Use `key={presentation.id}` prop on this component to let React reset it automatically — NO useEffect needed.
- Local state: `[jsonError, setJsonError]` = `useState<string | null>(null)`
- Local state: `[parsedSlides, setParsedSlides]` = `useState<SlideBlock[]>(presentation.slides ?? [])`
- `handleJsonChange`: update `jsonText`, attempt `JSON.parse`, on success set `jsonError = null` and `setParsedSlides(parsed)`, on failure set `jsonError = err.message`
