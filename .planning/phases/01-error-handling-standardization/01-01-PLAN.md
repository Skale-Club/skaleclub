  <action>
    Three changes to server/app.ts, all in this order:

    1. ADD import as the FIRST line of the file (before existing `import express`):
       ```typescript
       import 'express-async-errors';
       ```
       The existing line 1 (`import express, { type Request, Response, NextFunction } from "express";`) becomes line 2. This ordering is CRITICAL — express-async-errors patches Express's Router prototype at import time. If imported after express, async errors will still be silently swallowed.

    2. ADD ZodError import. Change the existing imports section to include:
       ```typescript
       import { ZodError } from "zod";
       ```
       Place it after the express-async-errors import and before the existing express import.

    3. REPLACE the entire error middleware block (current lines 61-67):
       ```typescript
       // Error handling middleware
       app.use((err: any, _req: Request, res: Response, _next: NextFunction) => {
         const status = err.status || err.statusCode || 500;
         const message = err.message || "Internal Server Error";
         res.status(status).json({ message });
         throw err;
       });
       ```
       With the fixed version:
       ```typescript
       // Error handling middleware
       app.use((err: any, _req: Request, res: Response, _next: NextFunction) => {
         // Safety: if response already started, delegate to Express default handler
         if (res.headersSent) {
           return _next(err);
         }

         // Zod validation errors → 400 with field-level details
         if (err instanceof ZodError) {
           return res.status(400).json({
             message: "Validation failed",
             errors: err.flatten().fieldErrors,
           });
         }

         // All other errors → use status + message
         const status = err.status || err.statusCode || 500;
         const message = err.message || "Internal Server Error";

         // Log server errors for debugging
         if (status >= 500) {
           console.error(err);
         }

         res.status(status).json({ message });
       });
       ```

    Key differences from original:
    - `throw err` REMOVED (was crashing the process — ERR-01 fix)
    - `headersSent` guard ADDED (prevents "Cannot set headers" errors)
    - `instanceof ZodError` branch ADDED (ERR-03: returns 400 with fieldErrors)
    - `console.error(err)` for 5xx errors ADDED (debugging aid)
    - ZodError branch comes BEFORE generic handler (order matters — first match wins)
  </action>
