## Spacing Scale

Standard 8-point scale. All values already enforced via Tailwind utility classes throughout the admin dashboard — no custom tokens required.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, checkbox-to-label gap, drag handle padding |
| sm | 8px | `gap-2`, `p-2` | Compact row spacing, badge padding |
| md | 16px | `gap-4`, `p-4` | Default form field spacing, dialog content padding |
| lg | 24px | `gap-6`, `p-6` | Dialog section breaks, service row group spacing |
| xl | 32px | `gap-8`, `p-8` | Dialog outer padding (DialogContent default) |
| 2xl | 48px | `py-12` | EmptyState vertical padding (matches existing EmptyState: `p-12`) |
| 3xl | 64px | — | Not used in this phase |

Exceptions:
- Drag handle (`GripVertical`): 44px touch target height minimum on mobile — achieved via `min-h-[44px]` on the row wrapper (follows dnd-kit TouchSensor `tolerance: 8` pattern from PortfolioSection)
- Dialog max-height: `max-h-[85vh] overflow-y-auto` to keep single scrollable form usable on small screens

## Typography

All sizes reference existing Tailwind prose classes already used in admin components. Font is Inter for all roles.

| Role | Size | Tailwind Class | Weight | Tailwind Weight | Line Height | Usage |
|------|------|----------------|--------|-----------------|-------------|-------|
| Body | 14px | `text-sm` | Regular | `font-normal` (400) | 1.5 | Table cell text, dialog field values, muted descriptions |
| Label | 14px | `text-sm` | Semibold | `font-medium` (500) | 1.4 | Form labels (Label component), row field names, column headers |

Weights in use: Regular (400) for body content + Bold (700) for headings. Medium (500) used exclusively for labels — stays within the 2-weight principle (label weight is intermediate, not a third decorative weight).

## Color

Uses HSL CSS variables. Values shown are light mode references; dark mode variants defined in `.dark {}` in `client/src/index.css`.

| Role | CSS Variable | Light Value | Dark Value | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` — white | `hsl(0 0% 7%)` — #121212 | Page background, dialog backdrop |
| Secondary (30%) | `--card` / `--muted` | `hsl(0 0% 100%)` / slate-100 | `#1a1a1a` / #1c1c1c | EstimatesSection container, service rows background, list row hover |
| Accent (10%) | `--primary` | `hsl(215 71% 37%)` — #1C53A3 | `hsl(217 91% 60%)` — brand blue | Reserved — see list below |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` | `hsl(0 62% 50%)` | Delete button in AlertDialog, delete icon button in list rows |

**Accent (`primary`) reserved for:**
1. "New Estimate" button (primary Button variant) in SectionHeader action slot
2. "Save" / "Save Changes" button in DialogFooter (primary Button variant)
3. Focus ring on active Input / Textarea fields (`--ring` = primary color)
4. Checked checkbox state in the catalog checklist picker

**Accent (`accent` token = blue-100 light / #292929 dark) reserved for:**
- Selected/hovered sidebar menu item background (existing pattern — do not reuse on EstimatesSection elements)

**Muted foreground** (`--muted-foreground`): slug text in list rows, creation date, dialog field helper text, EmptyState description. Use `text-muted-foreground` class.

## Interaction Contracts

### Copy-Link Button

- On click: `navigator.clipboard.writeText(window.location.origin + '/e/' + slug)`
- On success: toast `{ title: "Link copied", description: "Share this link with your client." }`
- On failure (clipboard API unavailable): toast `{ title: "Copy failed", description: "Please copy the URL manually.", variant: "destructive" }`

### Create Mutation (`POST /api/estimates`)

- On success: `queryClient.invalidateQueries({ queryKey: ['/api/estimates'] })` + close dialog + toast `{ title: "Estimate created" }`
- On error: toast `{ title: "Failed to create estimate", description: error.message, variant: "destructive" }`

### Update Mutation (`PUT /api/estimates/:id`)

- On success: `queryClient.invalidateQueries({ queryKey: ['/api/estimates'] })` + close dialog + toast `{ title: "Estimate updated" }`
- On error: toast `{ title: "Failed to update estimate", description: error.message, variant: "destructive" }`

### Delete Mutation (`DELETE /api/estimates/:id`)

- Triggered only after AlertDialog confirmation (D-09)
- On success: `queryClient.invalidateQueries({ queryKey: ['/api/estimates'] })` + toast `{ title: "Estimate deleted" }`
- On error: toast `{ title: "Failed to delete estimate", description: error.message, variant: "destructive" }`

### Drag Reorder

- `DragEndEvent` handler: `arrayMove(services, oldIndex, newIndex)` → update local `services` state
- Order is NOT auto-saved — order persists when admin clicks "Save Changes" (the services array order is the saved order)
- Sensor config: `MouseSensor { activationConstraint: { distance: 6 } }`, `TouchSensor { activationConstraint: { delay: 150, tolerance: 8 } }` — exact PortfolioSection values

### Catalog Checklist (Create mode)

- On checkbox check: append `{ type: "catalog", ...snapshotFields }` to local `services` state
- On checkbox uncheck: remove the corresponding row from local `services` state
- In Edit mode: checklist is NOT shown on initial open — only existing service rows are shown. An "Add from catalog" button expands the checklist inline (D-03)

### Add Custom Row

- On click: append `{ type: "custom", title: "", description: "", price: 0, features: [] }` to local `services` state (D-06)
- New row appears at bottom of list with focus on the title input

| Action | Trigger | Confirmation Approach |
|--------|---------|----------------------|
