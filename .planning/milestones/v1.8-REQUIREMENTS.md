# Requirements: Skale Club Web Platform - v1.8 Notification Templates System

**Defined:** 2026-05-04
**Core Value:** Admin can configure, preview, and edit all outbound notifications (Twilio SMS + Telegram) from a single panel using DB-stored templates with variable substitution — eliminating hardcoded message strings and enabling multi-channel delivery without code changes.

## v1.8 Requirements

### Schema & Templates Foundation

- [ ] **NOTIF-01**: A `notification_templates` table stores one row per (event × channel) pair — fields: `id`, `event_key` (e.g. `new_chat`, `hot_lead`, `low_perf_alert`), `channel` (`sms` | `telegram`), `body` (text with `{{variable}}` placeholders), `active` (boolean), `created_at`, `updated_at`.
- [ ] **NOTIF-02**: A shared notification dispatcher service (`server/lib/notifications.ts`) accepts an event key + variable map and fans out to every active channel using the matching template.
- [ ] **NOTIF-03**: The 3 existing hardcoded Twilio notification messages (`new_chat`, `hot_lead`, `low_perf_alert`) are migrated to DB seed templates — existing behavior is preserved exactly.
- [ ] **NOTIF-04**: Template variable substitution replaces `{{variable}}` tokens with runtime values; unknown variables render as empty string (no crash).
- [ ] **NOTIF-05**: Drizzle/Zod contracts for `notification_templates` are defined in `shared/schema/notifications.ts` and re-exported from `shared/schema.ts`.

### Telegram Integration

- [ ] **NOTIF-06**: Admin can configure a Telegram bot token and target chat ID via `/api/integrations/telegram` (GET + PUT), stored alongside the existing Twilio settings pattern.
- [ ] **NOTIF-07**: `server/integrations/telegram.ts` exports `sendTelegramMessage(config, text)` which posts to the Bot API — no external SDK required (native fetch).
- [ ] **NOTIF-08**: The shared dispatcher routes the same 3 notification events to Telegram when a template with `channel = 'telegram'` is active and Telegram is configured.
- [ ] **NOTIF-09**: Telegram messages support basic Markdown (bold names, newlines) — the body template renders cleanly in Telegram clients.

### Admin Notifications Panel

- [ ] **NOTIF-10**: Admin dashboard gains a `Notifications` section listing all notification events, each showing: event name, active channels (SMS / Telegram badges), and an Edit button.
- [ ] **NOTIF-11**: Clicking Edit opens an inline editor showing the template body for each channel — admin can edit the text, toggle the channel active/inactive, and save.
- [ ] **NOTIF-12**: Template editor displays available variables for each event as a reference list (e.g., `{{name}}`, `{{phone}}`, `{{company}}`).
- [ ] **NOTIF-13**: Saving a template persists to DB and the next triggered notification uses the new text without a server restart.
- [ ] **NOTIF-14**: Telegram integration settings (bot token, chat ID) are configurable from the existing Integrations section alongside Twilio.

## Out of Scope

| Feature | Reason |
|---------|--------|
| End-user / lead-facing notifications | Current scope is admin-alert notifications only |
| Email channel | Separate integration — different infrastructure |
| Notification scheduling / delays | Not requested — triggered on event only |
| Notification history / delivery log | Future milestone |
| Per-admin recipient routing | All notifications go to configured recipients |
