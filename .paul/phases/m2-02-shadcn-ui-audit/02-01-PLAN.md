# Plan M2-02-01 — Shadcn UI Base Audit

**Milestone:** 2 — Design + Structural Refactor
**Phase:** 2 — Shadcn UI Base Audit
**Status:** Ready for apply

---

## Goal

Audit and normalize every base UI component in `client/src/components/ui/` so that:
- No harsh solid borders (black/white/slate/gray hex)
- All borders use the `--border` token (soft alpha hairline)
- Defaults look correct in both light and dark themes without per-instance overrides
- Variants are consistent across the codebase

**Why this is first:** These components are consumed by every admin section and most of the public site. Fixing them at the source prevents reintroducing harsh borders in Phase 3 and every section refactor afterwards.

## Non-goals

- Do NOT change component APIs (props, variants stay the same)
- Do NOT add new variants or components
- Do NOT touch consumers (pages/sections) — that's Phase 3 onwards
- Do NOT change functional behavior (focus trap, keyboard nav, a11y)

---

## Risks

| Risk | Mitigation |
|------|-----------|
| Token reference broken (typo `border-border` vs `border-input`) | After each file, grep for remaining issues; run type check |
| Visual regression on a rarely-used component (e.g., Calendar) | Secondary sweep is audit-only; fix only if harsh border |
| Dark mode contrast too low after softening | If any element becomes invisible in dark, raise alpha from 6% to 10% — keep central |
