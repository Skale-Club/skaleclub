<objective>
## Goal
Eliminar o flash EN→PT adicionando um endpoint de preload que entrega todo o cache PT do DB de uma vez, e pre-aquecer o `translationCache` no client antes do primeiro render quando lang='pt'. Mudar o default de idioma de 'en' para 'pt'.

## Purpose
O backend de tradução funciona (Gemini + cache no DB). O problema é que o `translationCache` do client é em memória e começa vazio a cada page load. Cada string dinâmica dispara um batch de 50ms + round-trip à API, causando flash EN→PT visível. Com o preload, o cache já está cheio antes do primeiro render.

## Output
- `GET /api/translations/preload?lang=pt` — retorna `{ [sourceText]: translatedText }` para todo o cache PT
- `LanguageContext.tsx` — busca preload na inicialização quando lang='pt', popula `translationCache` antes de montar filhos
- Default language: `'en'` → `'pt'`
</objective>

    ```typescript
    app.get("/api/translations/preload", async (req, res) => {
      const lang = (req.query.lang as string) || "pt";
      const cached = await db
        .select({ sourceText: translations.sourceText, translatedText: translations.translatedText })
        .from(translations)
        .where(and(eq(translations.sourceLanguage, "en"), eq(translations.targetLanguage, lang)));

      const result: Record<string, string> = {};
      cached.forEach((row) => { result[row.sourceText] = row.translatedText; });
      res.json({ translations: result });
    });
    ```

    IMPORTANTE: Não requer autenticação — é leitura pública do cache.
    Importar `eq`, `and` já estão no arquivo. `translations` (schema) já está importado.

    2. Importar `translationCache` do hook de tradução para pre-aquecer.

       Adicionar export em `useTranslation.ts`:

    3. No `LanguageProvider`, adicionar `useEffect` que roda na montagem e quando `language` muda para 'pt':
       ```typescript
       useEffect(() => {
         if (language !== 'pt') return;
         fetch(`/api/translations/preload?lang=pt`)
           .then(r => r.json())
           .then(({ translations }) => {
             if (!translations) return;
             Object.entries(translations).forEach(([src, tgt]) => {
               translationCache.set(`pt:${src}`, tgt as string);
             });
             // Disparar evento para forçar re-render de todos os consumers
             window.dispatchEvent(new CustomEvent('translations-updated', {
               detail: { allDone: true },
             }));
           })
           .catch(() => {}); // falha silenciosa — t() ainda funciona via batch
       }, [language]);
       ```

    IMPORTANTE: O fetch acontece de forma assíncrona. Na primeira vez que componentes montam e `t()` é chamado, o cache pode ainda não estar pronto — isso é aceitável para a primeira visita (strings novas nunca antes traduzidas). Para strings JÁ no DB, o preload as entrega antes da janela de 50ms do batch disparar.

    Não usar `Suspense` nem bloquear a renderização — o preload é best-effort e o batch existente é o fallback.
