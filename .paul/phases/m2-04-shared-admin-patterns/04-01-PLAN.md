## Design / API

### 1. `AdminCard`

Purpose: Standard container for admin content blocks. Replaces the scattered `rounded-2xl border bg-card p-5 sm:p-6` pattern.

```tsx
<AdminCard padding="default" tone="default">
  ...content
</AdminCard>
```

Props:
- `padding`: `"compact" | "default" | "hero"` → `p-4` / `p-6` / `p-8`
- `tone`: `"default" | "muted" | "accent"` → `bg-card` / `bg-muted/40` / `bg-primary/5`
- `className`: passthrough

Base: `rounded-2xl border bg-card text-card-foreground transition-colors`

---

### 2. `SectionHeader`

Purpose: Page-level header used at the top of every admin section.

```tsx
<SectionHeader
  title="Portfolio"
  description="Drag to reorder, click to edit"
  action={<Button><Plus /> Add Service</Button>}
/>
```

Props:
- `title`: string
- `description?`: string
- `action?`: ReactNode (usually a primary button or link)
- `icon?`: ReactNode

Layout: `flex flex-col sm:flex-row items-start sm:items-center justify-between gap-4 mb-6`

---

### 3. `EmptyState`

Purpose: Unified empty state for lists, tables, grids.

```tsx
<EmptyState
  icon={<Briefcase />}
  title="No services yet"
  description="Create services to display on your portfolio page"
  action={<Button>Add First Service</Button>}
/>
```

Props:
- `icon?`: ReactNode (renders in a rounded muted circle)
- `title`: string
- `description?`: string
- `action?`: ReactNode

Layout: `flex flex-col items-center justify-center text-center p-12 rounded-2xl bg-muted/30 gap-4`

---

### 4. `FormGrid`

Purpose: Consistent responsive grid for form fields.

```tsx
<FormGrid cols={2}>
  <Field label="Title">...</Field>
  <Field label="Slug">...</Field>
</FormGrid>
```

Props:
- `cols`: `1 | 2 | 3` (defaults to `2`, mobile collapses to 1)
- `gap`: `"sm" | "default"` → `gap-3` / `gap-4`
- `className`: passthrough

Implementation: `grid grid-cols-1 [sm|md]:grid-cols-[N] gap-[X]`
