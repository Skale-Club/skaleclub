<objective>
## Goal
Remove every runtime reference to the legacy single-form endpoints `/api/form-config` (GET/PUT) and `/api/form-leads/progress` from both client and server. All callers switch to per-form endpoints (`/api/forms/slug/:slug/config`, `/api/forms/slug/:slug/leads/progress`, `/api/forms/:id`). Code-only change â€” zero schema change, fully reversible.

## Purpose
Closes the M3 compat-shim chapter: after M3-01 introduced the `forms` table and made both legacy and new endpoints coexist, only runtime code paths that read/write `company_settings.formConfig` remain. Retiring them here unlocks 05-02 (column drop + migration) without risk of a request hitting a removed column.

## Output
- Dead legacy endpoints deleted (3 Express routes + 1 shared route contract entry)
- Public `LeadFormModal` always slug-scoped; marketing pages pass `"default"`
- Admin Leads section's detail-sheet question labels resolve from the scoped form's config (no more global fetch)
- "Edit Form" button removed from Leads section (Forms section owns form editing since M3-02)
- `FormEditorContent` requires `formId` (no more fallback)
</objective>
